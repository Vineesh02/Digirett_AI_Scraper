## 📋 What This Scraper Does

This is a **complete Lovdata Pro web scraper** that:

1. ✅ Scrapes **ALL Sources of Law** categories (laws, regulations, court decisions)
2. ✅ Scrapes **ALL 39 Legal Areas** from Lovdata Pro
3. ✅ Creates structured **XML files** for each document
4. ✅ Uploads to **Supabase Storage** in batches of 20 files
5. ✅ **Deletes local XML files** after upload (saves disk space)
6. ✅ Stores **metadata** in Supabase database
7. ✅ Creates **final Excel file** at the end with 500-character content preview

## 🚀 Step-by-Step Setup Guide

### Step 1: Setup Supabase Database

1. **Go to [Supabase](https://supabase.com)** and create account
2. **Create new project**
3. **Go to SQL Editor**
4. **Execute this SQL** (create table):

```sql
-- Create lovdata_metadata table
CREATE TABLE IF NOT EXISTS lovdata_metadata (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  
  -- File information
  file_name TEXT NOT NULL UNIQUE,
  file_hash TEXT NOT NULL,
  file_size BIGINT,
  
  -- Supabase Storage
  storage_bucket TEXT,
  storage_path TEXT,
  public_url TEXT,
  
  -- Legacy compatibility
  file_storage_uri TEXT NOT NULL,
  
  -- Classification
  category TEXT NOT NULL,
  subcategory TEXT,
  
  -- Source
  source_url TEXT NOT NULL,
  
  -- Content preview (500 chars)
  content_preview TEXT,
  
  -- Metadata
  metadata JSONB
);

-- Create indexes
CREATE INDEX idx_file_name ON lovdata_metadata(file_name);
CREATE INDEX idx_file_hash ON lovdata_metadata(file_hash);
CREATE INDEX idx_category ON lovdata_metadata(category);
CREATE INDEX idx_subcategory ON lovdata_metadata(subcategory);
CREATE INDEX idx_created_at ON lovdata_metadata(created_at DESC);
CREATE INDEX idx_storage_path ON lovdata_metadata(storage_path);

-- Full-text search on content
CREATE INDEX idx_content_preview ON lovdata_metadata 
USING GIN(to_tsvector('norwegian', content_preview));
```

5. **Create Storage Bucket:**
   - Go to **Storage** in Supabase dashboard
   - Click **New Bucket**
   - Name: `lovdata-docs`
   - Make it **Public**

6. **Get your credentials:**
   - Go to **Settings** → **API**
   - Copy **Project URL** (looks like: `https://xxxxx.supabase.co`)
   - Copy **anon/public key** (starts with `eyJhbGc...`)

## 🔄 How Batch Upload Works

### Every 20 Documents:

1. ✅ Document scraped from Lovdata Pro
2. ✅ XML file created locally in `scraped_xml/`
3. ✅ Metadata saved to Supabase database
4. ✅ File added to batch queue

**When batch reaches 20 files:**

5. ☁️ All 20 files uploaded to Supabase Storage bucket
6. 🗑️ All 20 local XML files **DELETED**
7. 📊 Continue scraping...

**At the end:**

8. 📦 Any remaining files (< 20) uploaded
9. 📄 Final Excel file created with all metadata
10. ✅ Complete!

## 🐛 Troubleshooting

### Login Failed

**Error:** "Login failed"

**Solutions:**
1. Check credentials in `.env` file
2. Verify Lovdata Pro account is active
3. Try logging in manually in browser first
4. Check if account is locked

### No Files in Bucket

**Error:** Files not appearing in Supabase Storage

**Solutions:**
1. Verify bucket name is `lovdata-docs`
2. Check bucket is set to **Public**
3. Look in Supabase logs for upload errors
4. Verify `SUPABASE_KEY` has storage permissions

### Database Connection Failed

**Error:** "Database connection failed"

**Solutions:**
1. Check `SUPABASE_URL` and `SUPABASE_KEY` in `.env`
2. Verify table `lovdata_metadata` exists (run SQL)
3. Check project is not paused in Supabase
4. Use service_role key if anon key doesn't work

### Disk Space Full

**Error:** "No space left on device"

**Solution:**
- Reduce `BATCH_UPLOAD_SIZE` to 10 or 5
- Files upload and delete more frequently
- Less local storage needed

### Browser Won't Open

**Error:** Selenium fails to start

**Solutions:**
```bash
# Update Chrome driver
pip install --upgrade webdriver-manager

# Install Chrome browser
# Ubuntu/Debian:
sudo apt-get install google-chrome-stable

# Mac:
brew install --cask google-chrome
```
